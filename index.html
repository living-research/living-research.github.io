Living Research.

When AI makes text trivial, the value is in reproducible data and verifiable code. Research belongs in repositories, not documents.

The problem

  • Papers freeze knowledge into static PDFs that can't be verified, updated, or rerun.
  • Results sit behind paywalls while the code and data to reproduce them are missing.
  • Peer review happens behind closed doors between two anonymous reviewers.
  • By the time a paper is published, the field has already moved on.
  • In a 2016 Nature survey, over 70% of 1,576 scientists reported failing to reproduce another researcher's experiments.

The alternative

  • Data and code are the source of truth — not a narrative describing them.
  • Experiments rerun automatically via CI on every change.
  • Findings publish continuously to a living page, not a frozen PDF.
  • Git history is the intellectual record — every commit traces how thinking evolved.
  • Contributions happen through issues and PRs — transparent, attributable, open.
living-research/template
├── data/
│   ├── raw/            # immutable source data
│   └── processed/      # derived datasets, reproduced by CI
├── experiments/        # analysis scripts
├── docs/               # published findings via GitHub Pages
├── .github/workflows/  # CI that reruns experiments on every push
├── Makefile            # make run
└── requirements.txt

Principles

  • Reproducibility by default. If it can't rerun, it's not research.
  • Living, not frozen. New data comes in? Update the repo. Found a flaw? Open an issue. Knowledge evolves.
  • Show the work. Dead ends, pivots, negative results — the history is as valuable as the conclusion.
  • Credit through contribution. Git tracks who did what. No ambiguous author lists.
  • Text serves the data. Every sentence must add information not already present in the code or results.

The published page is a view of the research. The repo is the research.
